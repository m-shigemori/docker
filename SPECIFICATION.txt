============================================================
ContainerExecuter プロジェクト仕様書 (2026/05/06)
============================================================

1. プロジェクト概要
------------------------------------------------------------
本プロジェクトは、Dockerコンテナの管理をモダンかつ洗練されたGUIで
行うためのデスクトップアプリケーションです。
当初のPyQt5ベースからPyQt6への移行を経て、最終的にCustomTkinterを
採用し、高品質な視覚演出と高い操作性を実現しました。

2. アーキテクチャ設計
------------------------------------------------------------
DDD（ドメイン駆動設計）から、Web開発に馴染みのあるシンプルな3層構造へ
リファクタリングされました。

- app/models/   : データ構造（Containerクラス）の定義
- app/services/ : Docker操作の実ロジック（subprocess等）
- app/views/    : GUIの構築（CustomTkinter, UI管理）
- main.py       : アプリケーションのエントリーポイント（依存注入・起動）

3. GUI・視覚演出ロジック (コア機能)
------------------------------------------------------------
本アプリの最大の特徴は、背景画像とUIのシームレスな融合です。

A. 背景画像管理 (UIManager / BackgroundController)
   - ぼかし背景:
     背面にウィンドウ全体を覆う画像を配置。アスペクト比を維持しつつ
     ウィンドウの「長辺」を基準にリサイズ・クロップを行い、隙間を
     完全に排除します。さらに GaussianBlur を適用し、高級感を演出。
   - 前面メイン画像:
     右端に配置される鮮明な画像。ウィンドウの高さに合わせ、アスペクト比を
     維持したまま表示されます。

B. レスポンシブ・レイアウト
   - 絶対配置 (place) の採用:
     Tkinterの制約を回避するため、ピクセル単位での計算による配置を行い、
     画像の隙間や変形時の崩れを物理的に防いでいます。
   - スロットリング (0.1秒):
     リサイズ中の負荷を抑えるため、操作が止まってから100ms後に描画を
     実行するデバウンス処理を実装。
   - 動的フォントサイズ:
     ウィンドウ幅（860px～）に応じて、フォントサイズを12px～18pxの間で
     自動調整します。

C. 1:2:1 カラム比率
   - リスト表示（STATE: 1 / NAME: 2 / ACTION: 1）の比率を厳密に維持し、
     情報の視認性を確保しています。

4. 主要ロジックと機能
------------------------------------------------------------
- コンテナ一覧取得:
  docker ps コマンドの出力をパースし、Containerモデルのリストを生成。
- ライフサイクル管理:
  START / STOP / DELETE (コンテナ+イメージ) の各操作をサービス層で実行。
- ターミナル実行:
  gnome-terminal を通じてコンテナ内の /bin/bash を起動。
- モード切替:
  STANDARD（通常操作）と EDIT（削除専用）をワンクリックで切り替え。
  背景画像を維持したままリスト部分のみを高速に更新。

5. 開発者・メンテナンス向けノート
------------------------------------------------------------
- カラー指定: CustomTkinterの仕様上、rgba() ではなく HEX（16進数）を使用してください。
- サイズ変更: place() メソッドに width/height を渡さず、必ず configure() で
  サイズを確定させてから配置してください。
- 終了処理: CLOSEボタンは destroy() と sys.exit(0) を呼び、バックグラウンドに
  プロセスを残さないように設計されています。
